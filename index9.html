Project Idea – Location Triggered Alarm ‘Never Miss a Stop’

6.1 Overview

This project will aim to create a mobile application which has a location triggered alarm. The user will be able to choose a location from a map and set how far away from the target location the alarm will go off.
It will also contain features that will save the users’ alarms, allow certain customisation options for the alarms, offer online or offline option and operate in the background like regular stock alarms that come with all mobile devices.

6.2 Motivation

This would be an application that is targeted towards users who frequently travel on public transportation. With the busy and fast paced lifestyle of our everyday lives, moments on public transportation becomes a great time to catch up on some sleep.
As a result of catching up on some sleep, we may inadvertently miss our stop and thus waste valuable time and this can cause anxiety and stress that would impact the quality of sleep. This application aims to put these anxieties aside so the user can sleep knowing this application will wake them up when they are nearing their destination.

6.3 Description

This application in the simplest terms is an alarm with location tracking capabilities. The application will require access to the device’s time and date, GPS services and limited internet access. In order to minimise data usage, a map of the city of the user’s choosing will be installed when the application is opened for the first time, which will allow offline usage.

6.3.1 Key User Features

The most important thing from a user’s perspective is the ability to accurately set the destination, the timing of when the alarm triggers (i.e. the distance from destination for the alarm to trigger) and being able to customise each alarm they set up by changing the ringtone, vibration settings and distance to set off alarm.

6.3.2 Design and Layout

The application will open to a home screen, which will be a map with the user’s current location displayed. At the top of the screen will be:

  • A search bar where the user can find their destination and when they press search, they will be directed to the search result screen
  • On the left side will be a menu that can be accessed by swiping the left edge of the screen that will have buttons for viewing existing saved alarms, updating the installed map and settings

The search result screen will have a display in the top half of the screen showing the destination, and the bottom half of the screen will have, with scrolling function:

  • A drop box for the user to select how far away from the destination for the alarm to start going off (from 300m to 1km)
  • A text box to give the alarm a name
  • An option to select a ring tone from existing music files (it will default to the user’s current ring tone)
  • A checkbox for whether it should be on repeat

    • Note - If checked, new options will appear allowing the user to select which day(s) to repeat on and after what time to become active

  • Two buttons side by side, one to save the alarm, one to cancel and return to the previous screen

The saved alarms screen will be where the user will be able to view all their saved alarms, with each save file displaying its name, destination, distance to destination to ring, whether it repeats and the option to turn the alarm on/off. The user will also be able to make modifications to existing saved alarms by tapping the save file which will lead the user to the search result screen to make the changes and save them.
This screen will also offer the user the choice to delete any saved alarm by dragging it to a disposal area at the bottom of the screen.
The update maps button from the side menu will connect the device to a server that holds all the maps available to the users. A comparison will be made of the map version on the device against the version available on the server, if the versions do not match the user will be given a prompt to update the existing map. Prior to updating the map, if the device is not connected to Wi-Fi the user will be given a warning regarding excessive data usage.
The settings page will be where the user can choose their tracking method (GPS, network or both), view application version, patch notes and contact the developer should any issue arise while they are using the application.

6.4 Tools and Technologies

The tools and technologies required for the project are as follows:

Java Programming language and Eclipse Integrated Development Environment (IDE)

An Object-Orientated Programming (OOP) language which allows the developer to focus on what data they would need and how these data would interact with each other to produce the desired result. This would all be done on the Eclipse IDE which offers the developer various tools they would need to develop the application such as code editor, compiler and debugger.

Google Maps Platform

This gives us access to the functions Google Maps use, as well as customise it in a way to suit our needs. This will give the application the maps it would need to work, along with routing functions if the application wishes to expand to asking how the user is travelling for a more accurate timing of the alarm trigger.

6.5 Skills Required

The skills required to deliver the project are as follows:

Java Developer

Outstanding knowledge of the Java language will be required along with familiarity with the Eclipse IDE. This skill and knowledge will be the backbone of the application, with the code being able to access and use location data from the phone, access and trigger the device’s vibration (if set on) and ringtone when a pre-set distance from the destination.

SQL Developer

As there are plans to hold a database of maps which will be available for downloading, knowledge of SQL will be required to set up, access, retrieve and edit the data stored as well as maintaining the server. These will allow users of our application to have a reliable source of maps to download for offline use, allowing the app to be used in places where internet reception would be poor as well as reduce the amount of data used by the application.
Other related skills that would be expected from a SQL developer include preparing the documents to be put into the database (in this case the maps), memory management of the server, taking regular database backups and perform bug fixes.

UX Designer

In order to make the application look and feel good to use, knowledge and skill in UX designing will be necessary. The primary purpose of this skill is to make the application as easy to use as possible so it doesn’t form a barrier that prevents people from using it.
This skill will be used to designing the user interface (menus, tabs and widgets), the navigation buttons and search fields the application needs, identify and troubleshoot any UI issues and make any adjustments based on user feedback.

Business Analyst

The business analyst will document the requirements for the application. The requirements will document key features and behaviours that the application will need to fulfill for the end users.

Tester

A tester will test the application before it is released to the general public through the app store. The purpose is to ensure that the application is fully operational and meets the specifications listed in the business requirements.

6.6 Outcome

Assuming the project is successful, it will allow regular public transportation users to rest on their journey without having to worry about oversleeping and missing their stop. If this application gets enough following, it would be possible that the stock alarms that come with most mobile devices will look to incorporate a location trigger function and become the new standard, and this project will be the spark that brings about this change.
From a user’s perspective, the fully workable iOS or Android application would be downloadable from either the App Store or Google Play Store that will allow the use to:

  1. Set a destination they would like the alarm to go off based on the GPS location of the mobile device
  2. Set a how far out from the distance that the user would like the alarm to go off (1km to 200 metres) from the destination
  3. Select the route that will be followed to increase the accuracy of the alarm
  4. Enable the user to tone of the alarm or whether the mobile device will vibrate.
